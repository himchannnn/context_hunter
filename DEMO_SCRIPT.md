# 🎙️ Context Hunter 기술 시연 대본 (Detailed Technical Demo Script)

이 대본은 단순한 기능 시연을 넘어, **"어떻게 구현했는지", "설계와 어떻게 일치하는지"**를 설명하는 데 초점을 맞췄습니다.

---

## 🏗️ 시스템 아키텍처 요약 (Intro)
*   **Frontend:** React (SPA), Vite, Tailwind CSS
*   **Backend:** Python FastAPI (REST API), SQLAlchemy
*   **AI Engine:** Llama 3.1 8b (via API)
*   **Database:** SQLite (Local Demo) / MariaDB (Production Design)

---

## 🎬 시나리오 & 기술 설명

### 1. 오프닝 (서비스 소개)
*   **멘트:**
    > "안녕하십니까, 'Context Hunter' 팀입니다.
    > 저희 서비스는 **FastAPI와 React를 기반으로 구축된** 문해력 향상 웹 서비스입니다.
    > 설계 문서(HLD)에서 정의한 대로, 사용자의 입력을 AI가 실시간으로 분석하여 '의미적 유사도'를 판단하는 것이 핵심 기술입니다."

### 2. 게스트 로그인 (접근성 & 아키텍처)
*   **Action:** '게스트로 시작하기' 클릭.
*   **🔑 기술적 구현 (Technical Note):**
    *   **구현 방식:** 설계 단계에서는 게스트도 DB에 기록을 남길지 고민했으나, **DB 리소스 최적화**를 위해 **'휘발성 유저(Ephemeral User)'** 방식으로 구현했습니다.
    *   **로직:** 게스트 로그인 시 JWT 토큰은 발급되지만, User ID는 `-1`로 설정되며 DB에 저장되지 않습니다. 이를 통해 불필요한 데이터가 쌓이는 것을 원천 차단했습니다.
    *   **설계 비교:** HLD의 '게스트 모드 지원' 요구사항을 충족하되, 구현 단계에서 효율성을 높인 사례입니다.

### 3. 메인 화면 & 보안 (Security)
*   **Action:** 메인 화면 진입.
*   **🔑 기술적 구현 (Technical Note):**
    *   **보안:** 모든 API 통신은 **JWT (JSON Web Token)** 기반의 인증을 거칩니다. (LLD Slide 26 준수)
    *   **비밀번호:** 일반 회원가입 시 **PBKDF2-SHA256** 알고리즘으로 단방향 해싱하여 저장합니다. 이는 설계 문서의 보안 가이드라인을 정확히 준수한 것입니다.

### 4. 게임 플레이 & AI 판정 (Core Feature)
*   **Action:** 문제 풀이 동작 (오답 및 정답 시연).
*   **🔑 기술적 구현 (Technical Note):**
    *   **AI 모델:** 설계 문서(Slide 7, 22)에 명시된 대로 **Llama 3.1 8b 모델**을 활용합니다.
    *   **프로세스:**
        1.  사용자 입력 -> API 서버(FastAPI) 수신.
        2.  서버가 AI 모델에게 **'Few-Shot Prompting'** 기법으로 유사도 판단 요청.
            *(프롬프트 설계: "문장은 달라도 핵심 의미(Context)가 같으면 정답으로 인정하라"는 지침 포함)*
        3.  AI가 0~100점 사이의 유사도와 피드백을 JSON으로 반환.
    *   **정확도:** 단순 키워드 매칭이 아니라, 문맥(Context) 자체를 이해하므로 동의어나 비유적 표현도 정확히 채점합니다.

### 5. 결과 화면 & 공유하기 (New Feature)
*   **Action:** 결과 확인 후 '공유하기' 클릭.
*   **🔑 기술적 구현 (Technical Note):**
    *   **구현 방식:** 설계 문서(Slide 4)의 '소셜 공유 기능'을 구현하기 위해, 별도의 SDK 설치 없이 **Web Share API (`navigator.share`)** 표준 기술을 도입했습니다.
    *   **선정 이유:** 모바일 환경에서 인스타그램, 카카오톡 등 설치된 앱과 자연스럽게 연동하기 위함입니다. PC 환경에서는 자동으로 클립보드 복사(Fallback)로 전환되는 하이브리드 로직을 적용했습니다.

### 6. 랭킹 시스템 (Data & Optimization)
*   **Action:** 랭킹 리스트 확인.
*   **🔑 기술적 구현 (Technical Note):**
    *   **DB 구조:** `Guestbook` 테이블을 통해 점수와 Streak(연속 정답) 정보를 관리합니다.
    *   **설계 비교:** 설계 문서에서는 게스트의 랭킹 등록을 제한적으로 허용한다고 되어 있었습니다. 저희는 게스트도 닉네임만 입력하면 경쟁에 참여할 수 있도록 하여 **사용자 참여율(Engagement)**을 높이는 방향으로 구현을 확장했습니다.

### 7. 마무리 (Infra)
*   **멘트:**
    > "마지막으로 인프라입니다. 설계 문서상에는 Docker와 Nginx Reverse Proxy를 명시(Slide 25)했으나,
    > 금일 시연은 로컬 개발 환경(Dev Server)에서 구동 중입니다.
    > 실제 배포 가능한 수준의 **`docker-compose` 설정과 환경 변수 관리**는 이미 코드베이스에 모두 포함되어 있음을 말씀드립니다. 이상입니다."
