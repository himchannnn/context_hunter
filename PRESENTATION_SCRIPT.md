# 🎙️ Context Hunter 최종 시연 발표 대본 (Final Script)

이 대본은 **기술적 깊이(Technical Depth)**와 **설계의 타당성(Architectural Validity)**을 동시에 어필하도록 구성되었습니다. 회색 박스 안의 내용은 실제 멘트 예시이며, 불릿 포인트는 강조할 핵심 키워드입니다.

---

## 1. 🏗️ 오프닝 & 아키텍처 (Intro)

*   **포인트:** FastAPI + React 조합 및 설계 문서 준수 강조
*   **멘트:**
    > "안녕하십니까, 'Context Hunter' 팀입니다.
    > 저희 서비스는 **FastAPI와 React를 기반으로 구축된** 문해력 향상 웹 서비스입니다.
    > 설계 문서(HLD)에서 정의한 대로, 사용자의 입력을 AI가 실시간으로 분석하여 '의미적 유사도'를 판단하는 것이 핵심 기술입니다."

---

## 2. 👤 게스트 로그인 (Architecture & Optimization)

*   **Action:** '게스트로 시작하기' 클릭
*   **포인트:** 휘발성 유저 설계로 DB 리소스 최적화
*   **멘트:**
    > "먼저 접근성을 고려한 게스트 모드입니다.
    > 설계 단계에서 저희는 불필요한 데이터 적재를 막기 위해 **'휘발성 유저(Ephemeral User)'** 아키텍처를 채택했습니다.
    > 보시는 것처럼 로그인을 하면 JWT 토큰은 정상 발급되지만, User ID는 `-1`로 설정되어 데이터베이스에 영구 저장되지 않습니다. 이를 통해 가벼운 트래픽 처리가 가능합니다."

---

## 3. 🔐 메인 화면 & 보안 (Security)

*   **Action:** 메인 화면 진입
*   **포인트:** 표준 보안 규격 준수 (JWT, PBKDF2)
*   **멘트:**
    > "메인 화면입니다. 현재 모든 API 통신은 **JWT (JSON Web Token)** 기반의 인증을 거치고 있습니다.
    > 또한, 회원가입 시 비밀번호는 **PBKDF2-SHA256** 알고리즘으로 단방향 해싱되어 저장됩니다. 이는 시스템 설계 문서(LLD)의 보안 가이드라인을 엄격히 준수한 구현입니다."

---

## 4. 🧠 게임 플레이 & AI 전략 (Core & Strategy) ⭐ **중요**

*   **Action:** 문제 풀이 동작 (오답 및 정답 시연)
*   **포인트:** Llama 3.1 사용 + **하이브리드 배포 전략 (Deployment Strategy)**
*   **멘트:**
    > "핵심 기능인 문제 풀이입니다.
    > 사용자가 답을 입력하면 백엔드 서버는 **Llama 3.1 모델**에게 'Few-Shot Prompting' 기법으로 분석을 요청합니다.
    > 단순 키워드 매칭이 아니라, 문맥(Context)을 이해하여 0~100점 사이의 유사도를 도출해냅니다."
    > 
    > **(강조)**
    > "특히, 저희는 **'API First, On-Premise Ready'** 전략을 취했습니다.
    > 개발 효율성을 위해 클라우드 API(Groq)로 로직 검증을 먼저 수행했지만, 시스템은 **OpenAI 표준 인터페이스 규격**으로 설계되었습니다.
    > 따라서, 추후 학교 서버나 로컬 GPU 서버로 이관할 때 코드를 수정할 필요 없이, **환경 변수(Env)의 주소만 변경하면 즉시 전환**이 가능한 유연한 아키텍처를 갖추고 있습니다."

---

## 5. 📤 결과 & 공유 (UX & Web Standards)

*   **Action:** 결과 확인 후 '공유하기' 클릭
*   **포인트:** Web Share API (표준 기술) 사용
*   **멘트:**
    > "결과 화면입니다. 모바일 친화적인 경험을 위해 무거운 SDK 대신 **Web Share API 표준 기술**을 도입했습니다.
    > 모바일에서는 카카오톡/인스타로 즉시 연동되고, PC 환경에서는 자동으로 클립보드 복사로 전환되는 하이브리드 로직이 적용되어 있습니다."

---

## 6. 🏆 랭킹 시스템 (Data Extension)

*   **Action:** 랭킹 리스트 확인
*   **포인트:** 참여율 증대(Engagement)를 위한 기획 확장
*   **멘트:**
    > "마지막으로 랭킹 시스템입니다. `Guestbook` 테이블을 통해 점수와 연속 정답(Streak)을 관리합니다.
    > 초기 설계보다 기능을 확장하여, 게스트 유저도 닉네임만 있으면 랭킹 경쟁에 참여할 수 있도록 하여 사용자 참여율을 높이는 방향으로 개선했습니다."

---

## 7. 🏁 마무리 (Infrastructure) ⭐ **변경됨**

*   **포인트:** **Podman 도입 (환경 적응력)** & Fail-over Plan
*   **멘트:**
    > "마지막으로 인프라입니다. 초기에는 Docker 사용을 고려했으나, **학과 서버 환경에 최적화하기 위해 Podman으로 컨테이너를 배포**했습니다.
    > 저희 팀은 인프라 환경의 제약 사항을 유연하게 해결할 수 있는 역량을 갖추고 있으며, `docker-compose` 호환성을 활용해 빠르고 안정적인 배포를 구현했습니다.
    > 
    > 자체 서버 장애 시 즉시 클라우드 API로 우회할 수 있는 **Fail-over 시스템**도 확보되어 있습니다.
    > 이상으로 Context Hunter 기술 시연을 마치겠습니다. 감사합니다."
